# Tech Stack

## Overview

Conductor is a **Claude Code Plugin** that operates entirely through Markdown-based skill definitions and templates. It contains no runtime code—all functionality is delivered through structured prompts interpreted by Claude Code.

## Core Technologies

### Plugin System
- **Platform**: Claude Code Plugin System
- **Manifest**: `.claude-plugin/plugin.json`
- **Skills Format**: Markdown files (`SKILL.md`) with YAML frontmatter

### Content Format
- **Primary Language**: Markdown (GitHub-flavored)
- **Configuration**: JSON (plugin manifest, metadata files)
- **Templates**: Markdown files for code styleguides and workflow definitions

## Version Control & Release

### Version Control
- **System**: Git
- **Hosting**: GitHub (cloudaura-io/conductor-claude-code)

### Release Management
- **Tool**: Release Please
- **Versioning**: Semantic Versioning (SemVer)
- **Changelog**: Auto-generated from conventional commits

## Project Structure

```

│   ├── setup/SKILL.md

    └── code_styleguides/     # Language-specific guides
```

## Dependencies

This plugin has **no runtime dependencies**. It relies solely on:
- Claude Code's plugin loading system
- Claude Code's built-in tools (Read, Write, Edit, Bash, Glob, Grep)

## Artifacts Generated by Conductor

When Conductor is used in a project, it creates:

| Artifact | Format | Purpose |
|----------|--------|---------|
| `conductor/product.md` | Markdown | Product definition |
| `conductor/product-guidelines.md` | Markdown | Style and branding guidelines |
| `conductor/tech-stack.md` | Markdown | Technology documentation |
| `conductor/workflow.md` | Markdown | Development workflow rules |
| `conductor/tracks.md` | Markdown | Track registry |
| `conductor/tracks/<id>/spec.md` | Markdown | Track specification |
| `conductor/tracks/<id>/plan.md` | Markdown | Implementation plan |
| `conductor/tracks/<id>/metadata.json` | JSON | Track metadata |
